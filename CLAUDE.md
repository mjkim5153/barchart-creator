# BarChart 생성기기 프로젝트

## 환경 정보

- **OS**: Windows 11 Pro
- **작업 디렉토리**: C:\Users\admin\workspace\barchart creator

## 작업 방식

### 계획 모드 우선 원칙

[중요] 모든 비자명한 구현 작업을 시작하기 전에 **계획 모드**를 사용합니다.

**언제 계획 모드를 사용하는가:**
- 새로운 기능 구현 또는 상당한 규모의 변경
- 여러 파일에 영향을 미치는 작업
- 아키텍처 결정이 필요한 작업
- 구현 방식에 다양한 선택지가 있는 경우

**계획 모드 프로세스:**
1. 프로젝트 구조와 코드베이스 탐색
2. 기존 패턴과 아키텍처 분석
3. 구현 전략 설계 및 문서화
4. 사용자 승인을 위해 계획 제시
5. 승인 후 구현 진행

이를 통해 불필요한 작업을 줄이고 올바른 방향으로 진행하도록 보장합니다.

## 코드 스타일

- 간결하고 명확한 코드 작성
- 요청된 작업에만 집중 (과도한 리팩토링 금지)
- 기존 코드 패턴 및 관례 존중

## 커뮤니케이션

- 간결한 응답 선호
- 진행 상황에 대한 명확한 업데이트
- 필요시 의사결정을 위해 사용자에게 질문
