# 운항스케줄 바차트 웹 프로그램 기획

## 전체 화면 레이아웃

```
┌─────────────────────────────────────────────────────┐
│                   설정 영역 (상단)                     │
│  [날짜 선택] [항공사 콤보] [TOF 다중선택] [검색버튼]   │
├──────────────────┬──────────────────────────────────┤
│  정보 요약 영역   │       바차트 표출 영역              │
│  (왼쪽 패널)     │  Y축: 항공기 / X축: 00~24시         │
│                  │                                   │
└──────────────────┴──────────────────────────────────┘
```

---

## 1. 설정 영역 (상단)

### 구성 요소

| 구성 요소 | 설명 |
|-----------|------|
| 날짜 선택 | 조회일 단일 선택 (date picker 1개) |
| 항공사 선택 | 콤보박스, 국적사만 표시, 단일 선택만 가능 |
| TOF 선택 | `TOF` 컬럼 값 기준, 다중 선택 가능 (체크박스 또는 멀티셀렉트) |
| 기재 필터 | 콤보박스: 전체/인천기재/국내기재 (바차트 표출만 영향, 요약에 영향 없음) |
| 검색 버튼 | 데이터 수집 트리거 |
| 양식 다운로드 | 엑셀 업로드용 템플릿 다운로드 |
| 엑셀 다운로드 | 현재 데이터를 3시트(요약/바차트/RAW) 엑셀로 내보내기 |

### 데이터 수집 동작 방식

- **항공사**: 검색 시 `icao_airline=[""]` (전체 항공사) 고정으로 API 호출
- **공항(TOF)**: 검색 시 전체 공항 대상으로 데이터 수집
- 즉, **검색 버튼은 항상 전체 데이터를 다운로드**

```python
# 검색 버튼 클릭 시 내부 동작 예시
process_flight_schedule(
    start_date="20260318",  # 선택한 조회일
    end_date="20260318",    # 조회일과 동일 값 (단일 날짜 조회)
    icao_airline=[""],      # 무조건 전체 항공사
    icao_codes=[...],       # 전체 공항 코드 (고정)
    file_name="./data/flight_schedule_20260318.xlsx"
)
```

### 설정 값의 역할

| 설정 | 데이터 수집 영향 | 바차트 영향 | 요약 영향 |
|------|----------------|------------|----------|
| 날짜 | O (API 호출 파라미터) | O | O |
| 항공사 | X (전체 다운로드) | O (필터) | O (가동률) |
| TOF | X (전체 다운로드) | O (필터) | O (필터) |
| 기재 | X | O (필터) | X |

## 2. 바차트 표출 영역 (우측)

### 축 구성

- **Y축**: 항공기 등록번호 (`Acno`) — **여객기만 표시**
  - 여객기 판단 기준: `TOF` 컬럼에 `CGE`, `CGO`, `CGC` 값이 **없는** 항공기
- **X축**: 시간 (00:00 ~ 24:00)
  - 1시간 단위 주요 구분선 (진한 색)
  - 10분 단위 보조 구분선 (옅은 색)

### 표출 필터 조건

- 설정에서 선택한 **항공사**에 해당하는 스케줄 BAR만 표시
- 설정에서 선택한 **TOF(공항)** 에 해당하는 스케줄 BAR만 표시
- `Bt_idx = Y` 인 레코드만 사용 (중복 제거)

### 스케줄 BAR 표출

- BAR는 `STD_KST` ~ `STA_KST` 구간으로 표현
- BAR 내부에 편명(`Fltno`), 출발지-도착지(`Depstn`→`Arrstn`) 등 텍스트 표시

### 확대/축소 및 스크롤

| 입력 | 동작 |
|------|------|
| 마우스 휠 | Y축 스크롤 (세로 이동) |
| Ctrl+휠 / 핀치 | X, Y 동시 줌 |
| ArrowLeft/Right | 수평 스크롤 |
| Ctrl+ArrowLeft/Right | X축 축소/확대 |
| Ctrl+ArrowUp/Down | Y축 확대/축소 |
| X축 제한 | 시작일 00:00 ~ 종료일 24:00 범위 내 제한 |
| 축소 시 | ROW_HEIGHT * yZoom < 30px이면 텍스트 숨김, 툴팁은 유지 |

### 툴팁 표시 정보 (마우스 오버)

- `Fltno` (편명)
- `Depstn` → `Arrstn` (출발지 → 도착지)
- `STD_KST` ~ `STA_KST` (출발/도착 예정시각)
- `Blocktime` (분)
- `Status` (운항 상태)
- `Actype` (기종)
- `NAT` (자유 텍스트, 엑셀 업로드 필수 입력 — 필터/집계에는 관여하지 않는 툴팁 전용 컬럼)
- `MTTT` (정수, 엑셀 업로드 필수 입력 — NAT와 동일하게 툴팁 전용)
- 지연(RO-STD): 실적 출발(`RO_KST`)이 계획 출발(`STD_KST`)보다 늦은 경우의 지연시간(분),
  프론트에서 계산 (백엔드 컬럼 아님)

> `D_OPER`(값은 `Y` 또는 빈칸만 허용)은 엑셀 업로드 필수 컬럼으로 받지만 툴팁·바차트
> 어디에도 표출하지 않는다 — 업로드 데이터 보관 전용 컬럼.

## 3. 정보 요약 영역 (좌측 패널)

### 설정 값의 영향

| 설정 | 정보 요약 영향 |
|------|--------------|
| 항공사 | O (가동률 분석에 영향) |
| TOF | O (선택한 TOF 기준으로 집계) |

### 표시 대상

- **국적사만** 표시 (외항사 제외)
- 국적사 코드: KAL, AAR, JJA, TWB, JNA, ESR, ABL, ASV, EOK, PTA, APZ

### 표시 정보

#### (1) 가동률 분석

- 항공사 필터와 무관하게 **국적사 전체**를 테이블 형태로 표출
- 화물기 제외, 여객기만 집계
- 테이블 컬럼: 국적사 | 평균기재수 | 계획BT(분) | 가동시간(h)
- **계산식**: 가동시간 = 누적 계획BT / (일별 운영기재수 합산) / 60
  - 예: 3/18~3/19 검색 시 → 누적BT / (3/18 기재수 + 3/19 기재수) / 60
  - 평균기재수 = 일별 운영기재수 합산 / 일수
- **날짜 필터**: 드롭다운으로 특정일 선택 시 해당일만의 가동률 표시
  - "전체 기간" 선택 시 기간 전체 합산 표시

#### (2) GROUND TIME 분석 테이블

- 국내선/국제선 필터, 기종 등급(A~E) 필터 제공
- 항공사별 GT 구간 비중(%) 테이블
- 국내선 구간: <30, 30, 35, 40, 45, 50, >50 (분)
- 국제선 구간: <50, 50, 55, 60, 65, 70, >70 (분)
- 셀 값: 비중(%), 마우스 오버 시 운항편수 표시
- **연결편 없는 항공편**: GT가 계산되지 않는 첫 번째 편은 가장 큰 구간(>50/>70)에 포함
- **항공사 제목에 전체 운항편수 표시**: 예) "AAR(전체) - 123편"

#### (3) 노선별 BT(블록타임) 통계

- 노선(`Depstn`-`Arrstn`) 검색 기능 포함
- 항공사별 해당 노선의 BT 통계:
  - 평균 BT (분)
  - 최장 BT (분)
  - 최단 BT (분)
  - 운항 빈도 (횟수)

## 데이터 소스 참고

- 수집 함수: `func_ubkais.py` → `process_flight_schedule()`
- 데이터 규격: `.claude/rules/ubkiais_rule.md` 의 출력 컬럼 구조 준수
- 핵심 컬럼: `Bt_idx`, `TOF`, `Domestic_Intl`, `Carrier_Type`, `Status`
- `NAT`, `MTTT`는 크롤링(`func_ubkais.py`) 경로에는 존재하지 않고 엑셀 업로드
  전용으로만 필수 입력되는 컬럼 (툴팁 표시 전용, 필터/집계 로직 없음)
- `D_OPER`도 엑셀 업로드 전용 필수 컬럼(값은 `Y` 또는 빈칸만 허용)이지만, NAT/MTTT와
  달리 툴팁·바차트 어디에도 표출하지 않는다 (필터/집계 로직 없음, 업로드 데이터 보관 전용)
