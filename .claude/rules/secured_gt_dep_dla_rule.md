# 확보GT 기반 국내선 출발지연(DEP_DLA) 자동 조정 규칙

## 개요

바차트 드래그 재배정, 비운항/스케줄조정 시나리오, 정시율 최적화(`build_ontime_scenario.py`)
등으로 항공기 연결관계(Acno 체인)가 바뀔 때, 기존에는 연결지연(CNX_DLA)만 재계산되고
출발지연(DEP_DLA)·관제지연(ATC_DLA)은 원본값 그대로 고정 보존되었다. 이 문서는 여기에
추가되는 "확보GT 임계값 교차에 따른 국내선 DEP_DLA ±3분 조정" 규칙을 정의한다.

- 구현 위치: `data_processor.py::compute_scenario_delay_adjustments()`
- 적용 범위: 이 함수를 호출하는 모든 경로에 자동 반영됨 — 드래그 재배정
  (`/api/update-acno`), 비운항 시나리오(`/api/scenario/cancel-flights`), 스케줄 조정
  시나리오(`/api/scenario/reassign-chain`), 리셋, 저장/불러오기, `build_ontime_scenario.py`의
  스왑/이식 시뮬레이션(`simulate_swap`/`simulate_insert`) 전부.
- 관련 문서: [[barchart_display_rules]] 6번(확보GT/CNX_DLA/DEP_DLA 정의 및 툴팁 표시 규칙,
  이 규칙은 그 값들을 실제로 변형하는 계산 규칙), [[ontime_optimization_rules]] 3번
  (국내선 MTTT 35/60 동적 재계산 — 이 규칙과 별개 축으로 동시에 적용될 수 있음).

---

## 배경: 확보GT와 실제 Turn-Around Time(TAT)

이전편이 실제 도착(RI)한 시점부터 다음편이 출발(RO)하기까지 평균적으로 **40분**이
소요된다(실제 TAT). GT는 계획GT/확보GT/실제GT 3종이 있는데(정의는
[[barchart_display_rules]] 6번 참고), 이 규칙은 그중 **확보GT**만 사용한다:

```
확보GT = 해당편 STD_KST - 직전편 RI_KST
```

MTTT 35분인 국내선 운항편은, 확보GT가 충분(40분 이상)하면 승객 탑승을 3분 일찍
시작시키는 실무 관행이 있다. 반대로 확보GT가 빠듯해지면(40분 미만) 탑승을 3분 늦게
시작한다. 이 규칙은 그 관행을 시나리오 재계산에 반영한다.

## 규칙

어떤 편의 **국내선**(`Domestic_Intl == '국내선'`) 여부는 그 편 자신의 속성만 보고
판정한다(직전편이 국내선인지 국제선인지는 무관 — [[ontime_optimization_rules]] 3번의
"직전편 국제선이면 MTTT 60"과는 별개 축이다).

베이스라인(원본, `original_df`) 확보GT와 시나리오 반영 후(현재, 캐스케이드 포함) 확보GT를
비교해 **SECURED_GT_THRESHOLD_MIN(40분)**을 교차하면 DEP_DLA를 조정한다:

| 베이스라인 확보GT | 신규 확보GT | 조정 |
|---|---|---|
| < 40분 | ≥ 40분 | DEP_DLA -3분 (하한 0, 음수 불가) |
| ≥ 40분 | < 40분 | DEP_DLA +3분 |
| 교차 없음 (양쪽 다 <40 또는 양쪽 다 ≥40) | | 조정 없음 |

- 40분 정각은 "40분 이상"(짧지 않음)으로 분류한다 (`<` 연산 기준).
- 조정폭은 **SECURED_GT_DEP_DLA_STEP_MIN(3분)** 모듈 상수로 정의되어 있다
  (`data_processor.py`).
- 베이스라인에 직전편이 아예 없던 편(원본 체인의 첫 편)은 베이스라인 확보GT가
  정의되지 않으므로 이 규칙을 적용하지 않는다(교차를 판정할 기준이 없음). CNX_DLA/RO/RI는
  기존 로직대로 그대로 재계산된다.
- 원본 DEP_DLA가 결측(NaN)이면 조정하지 않는다(조정 기준이 없음).

### DEP_DLA 클램프와 RO/RI 이동량의 일치

DEP_DLA는 음수가 될 수 없다(원본 DEP_DLA가 0이면 -3을 적용해도 0으로 유지). 이때 실제
출발시각(RO_KST) 이동량도 원래 의도한 -3분이 아니라 **클램프 이후 실제로 적용된
차이**(`new_dep_dla - orig_dep_dla`)만큼만 반영한다. 그렇지 않으면 "DEP_DLA는 0→0으로
안 바뀌었는데 RO는 3분 당겨지는" 표시값-실제값 불일치가 발생하기 때문이다. 클램프가
발생하지 않는 일반적인 경우(원본 DEP_DLA ≥ 3) 이 보정은 결과에 영향을 주지 않는다.

### 캐스케이드 전파

이 조정은 편 단위로 독립적으로 판정되지만, RO/RI 이동은 기존 캐스케이드 메커니즘을
그대로 활용해 하위 체인에 자연스럽게 전파된다 — 어떤 편의 RI가 이 조정으로 이동하면,
바로 다음 편은 그 이동된 RI를 기준으로 자신의 확보GT를 판정하게 된다(별도 전파 코드
불필요).

## 엣지 케이스 요약

| 케이스 | 처리 |
|---|---|
| 원본에 직전편이 없던 첫 편이 시나리오로 새 연결편이 생김 | 조정 스킵(베이스라인 확보GT 미정의) |
| 확보GT가 정확히 40분 | "짧지 않음"(≥40)으로 분류 |
| 원본 DEP_DLA가 NaN | 조정 스킵, RO/RI 이동 없음 |
| 이 편은 국내선인데 직전편(상대)이 바뀜 | 이 편 자신의 Domestic_Intl만 판정 기준 |
| 캐스케이드로 여러 편이 연쇄 영향 | 각 편이 독립적으로 자신의 베이스라인/현재 확보GT만 비교, 하위 편은 상위 편의 이동분을 자동으로 이어받음 |
| DEP_DLA=0인데 -3 교차 발생(클램프) | DEP_DLA는 0 유지, RO/RI는 DEP_DLA로 인한 추가 이동 없음(CNX_DLA 변화분만 반영) |

## 상수 정의

- `SECURED_GT_THRESHOLD_MIN = 40` (`data_processor.py`)
- `SECURED_GT_DEP_DLA_STEP_MIN = 3` (`data_processor.py`)
